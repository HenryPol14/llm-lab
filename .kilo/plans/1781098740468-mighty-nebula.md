# План рефакторинга скриптов: numbering strategy

## Текущая ситуация

Скрипты используют префиксные номера (00-14):
```
00-bootstrap-remote.sh              # первичная настройка удаленного хоста
01-install-proxmox-tools.sh         # инсталл инструментов Proxmox
02-enable-iommu.sh                  # IOMMU/VFIO для GPU
03-configure-network.sh             # network bridge + firewall
04-download-cloud-image.sh          # download Ubuntu cloud image
05-create-cloudinit-template.sh     # создание VM template
06-download-cloud-image.sh          # (дублируется - нужно удалить? или нет)
07-create-cloudinit-template.sh     # (дублируется - нужно удалить? или нет)
08-create-llm-vm.sh                 # создание/обновление LLM VM
09-install-guest-runtime.sh         # установка Docker runtime
10-install-nvidia-toolkit.sh        # NVIDIA Container Toolkit
10-create-llm-vm.sh                 # (дублируется в run-all.sh)
11-create-monitoring-vm.sh          # создание/обновление monitoring VM
10-deploy-llm-stack.sh              # deploy Ollama + OpenWebUI
11-deploy-monitoring-stack.sh       # deploy Prometheus + Grafana (удален, заменен на 13?)
12-create-monitoring-vm.sh          # создание/обновление monitoring VM
13-deploy-monitoring-stack.sh       # deploy Prometheus + Grafana
14-deploy-nginx-proxy.sh            # deploy nginx reverse proxy
```

**Проблемы:**
1. Непоследовательная нумерация (пропуски: 06, 07, 11 в одних местах, есть в других)
2. Дублирование (04/06 download-cloud-image, 05/07 create-cloudinit-template)
3. `run-all.sh` ссылается на старые номера (10-create-llm-vm, 11-create-monitoring-vm)
4. Файлы и run-all.sh не синхронизированы

## Вариант 1: Сохранить номера (минимальные изменения)

**Принцип:** оставить 00-14 но привести к последовательности без пропусков

```
00-bootstrap-remote.sh
01-install-proxmox-tools.sh
02-enable-iommu.sh
03-configure-network.sh
04-download-cloud-image.sh
05-create-cloudinit-template.sh
06-create-llm-vm.sh
07-install-guest-runtime.sh (LLM)
08-install-nvidia-toolkit.sh (LLM)
09-deploy-llm-stack.sh
10-create-monitoring-vm.sh
11-install-guest-runtime.sh (monitoring)
12-deploy-monitoring-stack.sh
13-deploy-nginx-proxy.sh
14-setup-nft-rules.sh
```

**Плюсы:**
- Минимальные изменения
- Понятная последовательность
- Сохраняется иерархия "bootstrap → infra → LLM → monitoring"

**Минусы:**
- Нужно перенумеровать 14 файлов
- run-all.sh нужно обновить

## Вариант 2: Убрать номера, использовать имена

**Принцип:** скрипты выполняются по имени, не по номеру

```
bootstrap-remote.sh
install-proxmox-tools.sh
enable-iommu.sh
configure-network.sh
download-cloud-image.sh
create-cloudinit-template.sh
create-llm-vm.sh
install-guest-runtime-llm.sh
install-nvidia-toolkit-llm.sh
deploy-llm-stack.sh
create-monitoring-vm.sh
install-guest-runtime-monitoring.sh
deploy-monitoring-stack.sh
deploy-nginx-proxy.sh
setup-nft-rules.sh
```

**Плюсы:**
- Читаемость имен
- Не нужно следить за номерами
- Легко добавлять новые скрипты между существующими

**Минусы:**
- Очередность выполнения неочевидна
- Нужно менять `run-all.sh` на explicit order

## Вариант 3: Группировка с префиксами

**Принцип:** группировать по категориям с префиксами

```
infra-01-install-proxmox-tools.sh
infra-02-enable-iommu.sh
infra-03-configure-network.sh

vm-01-download-cloud-image.sh
vm-02-create-cloudinit-template.sh
vm-03-create-llm-vm.sh
vm-04-create-monitoring-vm.sh

deployment-01-install-guest-runtime-llm.sh
deployment-02-install-nvidia-toolkit-llm.sh
deployment-03-deploy-llm-stack.sh
deployment-04-install-guest-runtime-monitoring.sh
deployment-05-deploy-monitoring-stack.sh

proxy-01-deploy-nginx-proxy.sh
proxy-02-setup-nft-rules.sh
```

**Плюсы:**
- Ясная группировка
- Легко найти нужный скрипт
- Не нужно перенумеровать всю цепочку при добавлении

**Минусы:**
- Больше префиксов
- Нужно обновить run-all.sh

## Рекомендация: Вариант 3 (группировка)

**Обоснование:**
1. Сохраняет иерархию выполнения
2. Упрощает навигацию
3. Легче добавлять новые шаги
4. Имена говорят о назначении

**Скрипты в run-all.sh нужно заменить на explicit вызовы с порядком внутри группы:**

```bash
# Bootstrap
"${SCRIPT_DIR}/00-bootstrap-remote.sh"  # можно оставить как есть

# Infra
"${SCRIPT_DIR}/infra-install-proxmox-tools.sh"
"${SCRIPT_DIR}/infra-enable-iommu.sh"
"${SCRIPT_DIR}/infra-configure-network.sh"

# VM creation
"${SCRIPT_DIR}/vm-download-cloud-image.sh"
"${SCRIPT_DIR}/vm-create-cloudinit-template.sh"
"${SCRIPT_DIR}/vm-create-llm-vm.sh"
"${SCRIPT_DIR}/vm-create-monitoring-vm.sh"

# Deployment LLM
"${SCRIPT_DIR}/deployment-install-guest-runtime-llm.sh" "${LLM_IP:-}"
"${SCRIPT_DIR}/deployment-install-nvidia-toolkit-llm.sh" "${LLM_IP:-}"
"${SCRIPT_DIR}/deployment-deploy-llm-stack.sh" "${LLM_IP:-}"

# Verification
"${SCRIPT_DIR}/check-llm-vm-quick.sh" "${LLM_IP:-}" "${MONITORING_IP:-}"

# Deployment Monitoring
"${SCRIPT_DIR}/deployment-install-guest-runtime-monitoring.sh" "${MONITORING_IP:-}"

# Proxy
"${SCRIPT_DIR}/proxy-deploy-nginx-proxy.sh"
"${SCRIPT_DIR}/proxy-setup-nft-rules.sh"
```

**Дополнительно:**
- Обновить README.md с новыми именами
- Добавить `scripts/lib/` внутренние функции (если есть)

**Команда для миграции (для справки):**
```bash
# Переименовать все файлы по новой схеме
# Обновить run-all.sh
# Обновить bootstrap-remote.sh
# Обновить README.md
```
